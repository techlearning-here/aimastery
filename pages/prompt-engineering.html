üìã The Universal Prompt Framework

Use this framework for ANY AI tool:

[ROLE] + [TASK] + [CONTEXT] + [FORMAT] + [CONSTRAINTS] + [EXAMPLES]

1. ROLE (Who should the AI be?)

"You are an expert [profession/role] with [X years] of experience in [domain]"

Examples:

  • "You are a senior software engineer specializing in React"
  • "You are a marketing strategist for B2B SaaS companies"
  • "You are a patient and encouraging teacher for beginners"

2. TASK (What do you want?)

"Your task is to [specific action] that [achieves goal]"

Examples:

  • "Your task is to write a cold email that converts prospects into meetings"
  • "Analyze this code and identify performance bottlenecks"
  • "Create a 7-day social media content calendar"

3. CONTEXT (Why and for whom?)

"This is for [audience] who needs [outcome]. The background is [situation]"

Examples:

  • "This is for first-time founders who need to understand cap tables"
  • "I'm preparing for a client presentation next week. The client is skeptical about AI"
  • "My team is overwhelmed with manual data entry from customer emails"

4. FORMAT (How should it look?)

"Format the output as [structure]. Use [style/tone]"

Examples:

  • "Format as a bullet-pointed list with headers. Use professional but friendly tone"
  • "Provide a markdown table with 3 columns: Feature, Benefit, Example"
  • "Write in the style of technical documentation with code examples"

5. CONSTRAINTS (What are the limits?)

"Keep it [length]. Avoid [unwanted elements]. Must include [required elements]"

Examples:

  • "Keep it under 300 words. Avoid jargon. Must include actionable next steps"
  • "Don't use technical terms without explanation. Include at least 2 real examples"
  • "Maximum 5 bullet points per section. Each point should be one sentence"

6. EXAMPLES (Show what you mean)

"Here's an example of what I'm looking for: [example]"

Examples:

  • "Like this: '[sample output]'"
  • "Similar to how Apple describes their products‚Äîsimple, benefit-focused"
  • "Here are 2 examples of the tone I want: [examples]"

üéØ Advanced Techniques

Chain-of-Thought Prompting

Get better reasoning by asking AI to think step-by-step:

"Let's think through this step by step:
1. First, analyze the problem
2. Then, identify possible solutions
3. Finally, recommend the best approach with reasoning

Walk me through your reasoning at each step."

Few-Shot Learning

Provide examples to teach the AI:

"Convert casual messages to professional ones:

Input: "hey can we talk tmrw?"
Output: "Good morning! Would you be available for a discussion tomorrow?"

Input: "thanks for the help dude"
Output: "Thank you very much for your assistance."

Now convert: "need the report asap"
Output:"

Iterative Refinement

Build on previous outputs:

Prompt 1: "Write an outline for a blog post about remote work"
[Get output]

Prompt 2: "Great! Now expand section 3 into 3 detailed paragraphs"
[Get output]

Prompt 3: "Perfect! Now add statistics to support the claims"

üí° Quick Prompt Improvements

Example 1: Writing

‚ùå Before

‚úÖ After

"Write an 800-word blog post for busy professionals on 'The 5-Minute Productivity System.'

Include:

  • Hook with surprising statistic
  • Personal anecdote
  • 3 actionable techniques (each 200 words)
  • Tools/resources section
  • Motivational conclusion

Tone: Conversational, encouraging, practical"

Example 2: Analysis

‚ùå Before

"Analyze this data"

‚úÖ After

"You are a data analyst. Analyze this sales data for Q4 2025: [paste data]

Provide:

  1. 3 key trends with % changes
  2. Anomalies or unexpected patterns
  3. Comparison to industry benchmarks
  4. 2 actionable recommendations with expected impact

Format: Executive summary (max 400 words)"

üìä Prompt Quality Checklist

Before sending any prompt, check:

  • Specific: Is it clear exactly what I want?
  • Complete: Have I provided all necessary context?
  • Constrained: Have I set appropriate boundaries?
  • Formatted: Did I specify the desired output format?
  • Testable: Can I objectively evaluate if the output is good?
  • Actionable: Will the output be immediately useful?
